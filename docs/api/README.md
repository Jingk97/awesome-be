# API 文档目录

本目录用于管理接口协定与模块文档。各模块 `README.md` 作为索引，再指向具体接口文档。

## 目录约定

- 每个模块使用独立子目录，例如 `docs/api/system/`。
- 每个模块必须有一个 `README.md` 用作索引。
- 每个接口使用单独的 Markdown 文件。

## 模板

- 模块模板：`docs/api/_template/module.md`
- 接口模板：`docs/api/_template/endpoint.md`

## 接口规范说明

### 基础约定

| 项目     | 规范                              |
| -------- | --------------------------------- |
| 基础路径 | `/api/v1`                         |
| 数据格式 | `application/json`                |
| 字符编码 | `UTF-8`                           |
| 时间格式 | Unix 毫秒时间戳（数字类型）       |
| 布尔值   | `true` / `false`（JSON 原生布尔） |

### 统一响应结构

所有接口均返回以下结构：

```JSON
{
  "code": 0,
  "message": "ok",
  "data": {},
  "request_id": "request-id"
}
```

| 字段         | 类型        | 说明                      |
| ------------ | ----------- | ------------------------- |
| `code`       | integer     | 业务状态码，0 表示成功    |
| `message`    | string      | 状态描述，成功时为 "ok"   |
| `data`       | object/null | 业务数据，失败时可为 null |
| `request_id` | string      | 请求唯一 ID，用于日志追踪 |

### 通用错误码

  | 错误码范围 | HTTP 状态码 | 说明       |
  | ---------- | ----------- | ---------- |
  | 0          | 200         | 成功       |
  | 1xxx       | 500         | 系统错误   |
  | 400x       | 400         | 参数错误   |
  | 401x       | 401         | 认证错误   |
  | 403x       | 403         | 权限错误   |
  | 404x       | 404         | 资源不存在 |
  | 409x       | 409         | 冲突错误   |
  | 429x       | 429         | 限流错误   |
  | 5xxx       | 500         | 服务器错误 |
